# 🎨 Theme Manager — 디자인 시스템 테마 관리

이 명령어는 포트폴리오 프로젝트의 **테마 설치·관리·상태 확인**을 총괄하는 메인 명령어입니다.

⚠️ **사전 조건**: `/project-init` 명령어가 먼저 실행되어야 합니다.

---

## 사용 가능한 테마 (동적 스캔)

실행 시 `.claude/commands/` 디렉토리에서 `2[a-z]*.md` 패턴의 파일을 Glob으로 스캔하여 테마 목록을 동적으로 구성합니다.

**스캔 방법:**
1. `.claude/commands/2[a-z]*.md` 파일들을 Glob으로 검색
2. 각 파일의 frontmatter(`description`)와 내용에서 테마 이름, 라우트, 강조색을 추출
3. 추출된 목록으로 사용자에게 선택지 제시

> 테마가 추가/삭제되면 `2x.*.md` 파일만 추가/삭제하면 자동 반영됩니다.

---

## 동작 분기 ($ARGUMENTS 기준)

### `$ARGUMENTS`가 **테마 이름**인 경우

해당 테마의 하위 명령 파일을 읽고 그 지시에 따라 테마를 설치합니다.

**실행 순서:**

1. **공용 인프라 확인** — `src/styles/provider.tsx`와 `src/styles/styled.d.ts`가 없으면 아래 "공용 인프라 파일" 섹션을 먼저 설치합니다.
2. **해당 테마 명령 파일 찾기** — `.claude/commands/2[a-z]*.md` 파일들을 Glob으로 스캔하여, `$ARGUMENTS`와 매칭되는 파일을 찾습니다. (파일명 또는 description에서 테마 이름 매칭)
3. **명령 파일의 Steps를 그대로 실행** — 파일에 정의된 단계를 순서대로 수행합니다.
4. **빌드 확인** — 아래 "빌드 확인" 섹션 실행.
5. **완료 보고** — 아래 "완료 보고 형식" 출력.

### `$ARGUMENTS`가 **`all`**인 경우

`.claude/commands/2[a-z]*.md` 파일을 스캔하여 발견된 모든 테마를 파일 이름 순서(2a → 2b → 2c → …)대로 설치합니다.

### `$ARGUMENTS`가 **`status`**인 경우

현재 테마 설치 상태를 점검하고 보고합니다.

**점검 항목:**

1. 공용 인프라 파일 존재 여부: `provider.tsx`, `styled.d.ts`, `globals.css`
2. `.claude/commands/2[a-z]*.md` 파일을 스캔하여 등록된 테마 목록 추출
3. 각 테마별로 토큰 파일(`src/styles/themes/{테마}.ts`)과 라우트 layout(`src/app/{테마}/layout.tsx`) 존재 여부 확인

**출력 형식:**

```
🎨 테마 설치 상태
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔧 공용 인프라
  ✅/❌ provider.tsx
  ✅/❌ styled.d.ts
  ✅/❌ globals.css

📦 테마 토큰 & 라우트 (2x.*.md 스캔 결과)
  ✅ {테마명} — themes/{테마}.ts + app/{테마}/layout.tsx
  ❌ {테마명} — 미설치 → /theme-on {테마명}
  ...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### `$ARGUMENTS`가 **비어 있거나 없는** 경우

사용자에게 테마 선택지를 안내하고 어떤 테마를 설치할지 질문합니다.

## 공용 인프라 파일 (최초 1회 생성)

아래 3개 파일은 **모든 테마가 공유**하는 인프라입니다.
테마 설치 전 존재 여부를 확인하고, 없으면 생성합니다.
이미 존재하면 **덮어쓰지 말고** 내용을 비교 후 필요한 부분만 업데이트합니다.

### `src/styles/styled.d.ts` — DefaultTheme 타입 확장

> ⚠️ 이 파일이 없으면 `props.theme.colors.*` 사용 시 TypeScript 빌드 오류 발생

### `src/styles/provider.tsx` — 공용 SSR StyleProvider

```typescript
'use client';

import React, { createContext, useContext, useEffect, useState } from 'react';
import { useServerInsertedHTML } from 'next/navigation';
import { ServerStyleSheet, StyleSheetManager, ThemeProvider } from 'styled-components';
import type { DefaultTheme } from 'styled-components';

// SSR 스타일 주입 (FOUC 방지) — Next.js App Router + styled-components v6 필수
export function StyleProvider({
  children,
  theme,
}: {
  children: React.ReactNode;
  theme: DefaultTheme;
}) {
  const [styledComponentsStyleSheet] = useState(() => new ServerStyleSheet());

  useServerInsertedHTML(() => {
    const styles = styledComponentsStyleSheet.getStyleElement();
    styledComponentsStyleSheet.instance.clearTag();
    return <>{styles}</>;
  });

  const content = typeof window !== 'undefined' ? (
    <ThemeProvider theme={theme}>{children}</ThemeProvider>
  ) : (
    <StyleSheetManager sheet={styledComponentsStyleSheet.instance}>
      <ThemeProvider theme={theme}>{children}</ThemeProvider>
    </StyleSheetManager>
  );

  return content;
}
```

## 공통 Rules (모든 테마 적용)

- **테마 일관성**: 모든 컴포넌트는 `props.theme.colors.*`, `props.theme.radius.*` 참조
- **`provider.tsx`, `styled.d.ts`**: 이미 존재하면 덮어쓰지 말고 내용을 비교 후 필요한 부분만 업데이트
- **섹션 컴포넌트**: `src/components/sections/`를 수정하지 않음 — `page.tsx`에서 import
- **빌드 성공**: 반드시 빌드 실행 후 에러 없음 확인
